## 分层结构描述

### 1. CLI 层 (`src/cli/`)

```
cli/
├── index.ts                # yargs 入口 (注册所有命令)
├── effect-cmd.ts           # Effect 风格的命令包装器
├── bootstrap.ts            # 传统 bootstrap 封装
├── ui.ts                   # 终端 UI 辅助 (ANSI 颜色/logo)
├── logo.ts                 # ASCII logo
├── heap.ts                 # 堆快照监控
├── network.ts              # 网络选项
├── error.ts                # 错误格式化
├── upgrade.ts              # 升级逻辑
├── cmd/                    # 命令实现
│   ├── run.ts              # run 命令 (核心入口)
│   ├── tui/                # TUI 应用 (SolidJS + Ink)
│   ├── run/                # run 交互式运行时
│   ├── debug/              # 调试子命令
│   ├── account.ts          # 账户管理
│   ├── agent.ts            # Agent 配置
│   └── ...                 # 其他命令
└── effect/                 # CLI 层 Effect 工具
    └── prompt.ts
```

#### 命令注册模式

- **`effectCmd`**: 推荐方式，返回 Effect 效果，自动处理 Instance 生命周期
- **`cmd`**: 传统 CommandModule 对象，适用于简单命令（`generate`, `uninstall`）

### 2. 核心运行时层 (`src/effect/` 和 `src/project/`)

```
effect/
├── app-runtime.ts       # 应用级 Effect Runtime (~40个服务合并)
├── run-service.ts       # 运行时创建工具 (makeRuntime, attach)
├── instance-state.ts    # 实例状态管理 (ScopedCache 按目录隔离)
├── instance-ref.ts      # InstanceRef / WorkspaceRef 标签
├── instance-registry.ts # 实例注册表 (dispose 管理)
├── bootstrap-runtime.ts # 启动时运行时
├── bridge.ts            # Effect Bridge
├── config-service.ts    # 配置服务
├── runner.ts            # Runner
└── runtime-flags.ts     # 运行时标志

project/
├── bootstrap.ts         # 实例启动编排
├── bootstrap-service.ts # 启动服务接口
├── instance-store.ts    # 实例存储 (加载/重载/释放)
├── instance-context.ts  # 实例上下文 (directory, worktree, project)
├── instance-layer.ts    # 实例层组合
├── instance-runtime.ts  # 实例运行时
├── project.ts           # 项目模型
└── vcs.ts               # VCS (Git) 集成
```

#### 实例生命周期

```
Load Input (directory)
    │
    ▼
InstanceStore.load()
    │
    ├── 检查缓存: 存在? → 等待已有 deferred
    │
    └── 创建新实例:
        ├── 创建 Deferred
        ├── fork 后台加载任务
        └── InstanceBootstrap.run():
            ├── Config.get()
            ├── Plugin.init()
            ├── Reference.init()
            ├── LSP.init()
            ├── ShareNext.init()
            ├── Format.init()
            ├── Vcs.init()
            ├── Snapshot.init()
            └── Project.init()
            (以上并行: unbounded concurrency)
                │
                ▼
        返回 InstanceContext
            { directory, worktree, project }
```

## 核心数据流

```
用户输入 (CLI)
    │
    ▼
yargs 解析 + effectCmd 包装
    │
    ▼
AppRuntime.runPromise(effect)
    │
    ▼
InstanceStore.load(directory)
    │
    ▼
InstanceBootstrap.run()  ──→  Config / Plugin / LSP / ...
    │
    ▼
Session 创建/恢复
    │
    ▼
SessionProcessor 循环:
    ├── Prompt 构建 (系统提示词 + 工具定义 + 历史)
    ├── LLM 调用 (streamText)
    ├── 工具执行 (ToolRegistry + MCP + Plugin)
    ├── 结果处理
    └── (循环直到 stop / compact)
    │
    ▼
输出: 标准输出 / TUI 滚动缓冲区 / JSON / WebSocket
```

## 设计模式总结

| 模式 | 实现 | 用途 |
|------|------|------|
| **Effect 依赖注入** | `Context.Service`, `Layer`, `Effect.gen` | 所有服务模块的解耦 |
| **InstanceState** | `ScopedCache<string, A>` 按目录缓存 | 每个项目目录独立的 service 实例 |
| **事件驱动** | `GlobalBus` (EventEmitter) | 跨模块通信 (实例释放、会话状态变更) |
| **V1/V2 共存** | `@opencode-ai/core/v1` vs `@opencode-ai/core/session` | 渐进式迁移 |
| **Drizzle ORM** | SQLite 表定义 + 查询 | 持久化会话、项目、账户等 |
| **AI SDK** | `vercel/ai` `streamText`, `wrapLanguageModel` | 统一的 LLM 接口 |
| **命令模式** | `effectCmd` 包装 yargs | CLI 命令的 Effect 化 |
| **Reducer 模式** | `SessionData` reducer | 增量更新 TUI 状态 |
| **ScopedCache** | 带作用域的缓存 | 实例级状态隔离和自动清理 |
